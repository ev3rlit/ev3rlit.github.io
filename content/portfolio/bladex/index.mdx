## 프로젝트 소개

BladeX는 제가 처음으로 참여한 게임 서버 프로젝트입니다. 신규 프로젝트의 게임 서버 아키텍처 설계부터 정식 런칭까지 전체 개발 사이클을 경험했습니다.

이 프로젝트에서 겪은 시행착오는 이후 프로젝트의 설계 방향을 결정짓는 중요한 교훈이 되었습니다.

## 주요 개발 내용

- Redis Sorted Set 기반 실시간 길드 랭킹 시스템
- 중복 로그인 처리 UX 개선
- 게임 핵심 시스템 (인벤토리, 우편함, 캐릭터, 상점, 길드, 인증)
- Jenkins CI/CD 파이프라인 구축

## 배운 점

이 프로젝트는 많은 **시행착오**를 경험한 프로젝트입니다:

1. **에러 추적 체계의 중요성**: 에러 추적 없이 운영하면 대응 비용이 기하급수적으로 증가
2. **트랜잭션의 필요성**: 데이터 정합성 이슈와 수동 DB 수정의 위험성 체감
3. **로그 체계 설계**: 체계적인 로그 없이는 운영팀이 개발자에게 의존

이 경험을 바탕으로 다음 프로젝트(삼국 블레이드 키우기)에서는 초기 설계 단계부터 에러 핸들링, 트랜잭션, 로그 체계를 우선 구축했습니다.
